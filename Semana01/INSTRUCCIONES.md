# INSTRUCCIONES DEL PORTAFOLIO - CC112

## Objetivo

Este repositorio será tu **portafolio personal de aprendizaje** durante el curso.

Aquí deberás evidenciar:

* Tu progreso semanal
* Tu mejora continua
* Tu forma de pensar al programar

Este repositorio no solo evalúa resultados, sino tu proceso.

---

## 1. Organización del repositorio

Debes organizar tu portafolio por semanas.

Estructura mínima esperada:
```text
semana01/
semana02/
semana03/
...
semana16/
```

Dentro de cada semana debes incluir:

* laboratorio/ → código desarrollado
* teoria/ → apuntes o ejercicios conceptuales
* reflexion.md → análisis personal

Ejemplo:

```text
semana03/
├── laboratorio/
├── teoria/
└── reflexion.md
```

---

## 2. Reflexión semanal (OBLIGATORIA)

Cada semana debes crear un archivo:

reflexion.md

Contenido mínimo:

* Qué aprendiste
* Qué te resultó difícil
* Cómo resolviste los problemas
* Qué mejorarás la siguiente semana

---

## 3. Código (buenas prácticas)

Tu código debe:

* Estar correctamente indentado
* Usar nombres descriptivos
* Evitar código innecesario
* Estar modularizado (usar funciones)

---

## 4. Uso de Git

Debes usar Git de manera continua.

* Al menos 2 commits por semana

---

## 5. Convención de commits

Formato obligatorio:

tipo: descripción breve

### Tipos permitidos:

* feat → nueva funcionalidad
* fix → corrección de errores
* refactor → mejora interna del código
* docs → documentación

---

### Ejemplos correctos:

```text
feat: semana03 laboratorio completo
fix: error en cálculo de factorial
refactor: mejora en uso de funciones
docs: reflexión semana02
```
---

### Ejemplos incorrectos:

```text
tarea lista
avance
cambios
```

---

## 6. Progreso del estudiante

Se espera que tu repositorio refleje:

* Mejora en la calidad del código
* Mejor organización con el tiempo
* Mayor claridad en tus soluciones

---

## 7. Reglas importantes

Está prohibido:

* Subir todo en un solo commit
* Copiar código sin citar fuente
* No incluir reflexiones
* Subir código que no compile

---

## 8. Evaluación

Se evaluará:

* Organización del repositorio
* Calidad del código
* Uso de Git
* Reflexión y mejora continua

---
